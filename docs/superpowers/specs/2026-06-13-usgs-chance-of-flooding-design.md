# Chance of flooding for USGS gauges â€” design

## Goal

Extend the existing "Chance of flooding" feature to the USGS gauges that carry
NOAA HEFS 5-day / 10-day probabilistic forecast data. Today the feature only
covers SVPA gauges, whose probability is derived by linear regression from a USGS
predictor gauge. The USGS gauges themselves (the predictors and other HEFS
points) show no chance because they aren't in the generated constants.

## Background

- `useFloodProbability(gage)` returns a `FloodChanceResult | null`. It returns
  null for any gauge not in `floodPredictionConstants.json`'s `gauges` array.
- The card row, the list/details status-pill badge, and the alert map pins all
  already read `useFloodProbability` / `useFloodRiskLevel`, so they surface a
  badge automatically once the hook returns a value.
- `computeFloodProbability({ p99, quantiles, ratingTable })` builds an
  exceedance-height curve from a gauge's rating table + HEFS quantiles and
  returns `heightToProbability(curve, p99)` for the greater of the 5/10-day
  windows. For a direct USGS gauge the "threshold stage" is simply the gauge's
  own red stage.

## Decisions (confirmed)

- **Scope:** every USGS gauge that publishes HEFS map-quantiles (verified live):
  Falls `USGS-38`/SQUW1, Carnation `USGS-22`/CRNW1, Monroe `USGS-SH5`/MROW1,
  South Fork `USGS-SF17`/GARW1, Middle Fork `USGS-MF11`/TANW1, North Fork
  `USGS-NF10`/SNQW1. Duvall `USGS-9` has no NWRFC HEFS point â†’ excluded. The
  headwaters metagage (`USGS-SF17/USGS-NF10/USGS-MF11`) is a synthetic flow-sum
  with no single NOAA site â†’ excluded; its component forks are covered directly.
- **Computation:** forecast-only â€” P(reach red stage) from the HEFS 5/10-day
  exceedance curves, greater of the two windows. No observed/nowcast blend (that
  needs per-gauge p50/p90/p99 we don't have for USGS gauges).
- **Threshold:** the app red stage (`gage.redStage` from `locationInfo`),
  consistent with the Flooding status pill, the map-pin color, and the
  at/above-red-stage gating used elsewhere.
- **UI:** reuse the existing surfaces (card row, list badge, map pin). No new UI.
- **Config location:** `floodPredictionConstants.json` is generated by an
  analytics system and must not be edited. The hand-maintained direct-gauge
  registry lives in a separate file.

## Components & changes

### 1. Direct-gauge registry â€” new `src/services/floodPrediction/directGauges.ts`

A typed array of `{ gaugeId, usgsSiteId, noaaSiteId, name }` for the six covered
gauges, plus `getDirectGaugeConstants(locationId)`. Hand-maintained, kept
separate from the generated constants. The four new NOAA site IDs (MROW1, GARW1,
TANW1, SNQW1) live here.

### 2. Service â€” `floodPredictionService.ts`

- Extend `getFloodProbability(locationId, redStage?)`:
  - SVPA gauge (in generated constants) â†’ unchanged (uses regression `p99`).
  - Direct USGS gauge (in the new registry) â†’ require `redStage`; fetch its own
    rating table (`usgsSiteId`) + map-quantiles (`noaaSiteId`) and call
    `computeFloodProbability({ p99: redStage, quantiles, ratingTable })`. Returns
    null when `redStage` is missing.
  - Reuses the existing rating/quantile caches.

### 3. Hook â€” `useFloodProbability.ts`

- A gauge is "covered" if it's an SVPA gauge **or** a direct USGS gauge.
- SVPA path unchanged (forecast + observed blend).
- Direct path: forecast-only. The forecast effect calls
  `getFloodProbability(locationId, gage.redStage)`; `observedProbability` is null;
  result is `combineFloodChance({ forecast, observedProbability: null })`.
- `gage.redStage` joins the effect deps for the direct path.

### 4. UI

No changes. Existing consumers light up once the hook returns a value. Direct
gauges can only reach the forecast buckets (`low` / `percent` / `veryHighClamp`),
never the observed-only `veryHigh` / `nearCertain` â€” already handled by the
label logic.

## Correctness check (during implementation)

Verify each covered gauge's `locationInfo.redStage` is populated and falls within
its rating table's gage-height range â€” i.e. red stage is gage-height feet, the
same datum as the exceedance curve. A gauge whose red stage is missing or out of
range yields no badge rather than a wrong one (graceful).

## Mock-replay note

`getFloodProbability` fetches HEFS directly (outside the mock-replay API seam), so
under a replay scenario the USGS chance reflects live forecasts, not the
time-shifted history â€” the same characteristic the SVPA forecast half already
has. Out of scope here.
